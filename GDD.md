# CCCraft — ゲームデザインドキュメント (GDD)

> **目的**: このドキュメントはClaude Codeによるvibe coding開発の仕様書として機能する。  
> **原則**: ここに書かれていないことはClaude Codeが自由に判断してよいが、書かれていることは厳守する。

---

## 1. ゲームビジョン

**一言で**: 空に浮かぶボクセルの島々を、仲間と一緒にグライダーで飛び回り、探索・建築・戦闘するオープンワールドゲーム。

**参考タイトルと取り入れる要素**:
- **Minecraft**: ボクセルワールド、自由なブロック設置/破壊、素材収集、クラフト
- **ゼルダの伝説 BotW/TotK**: 3rdパーソンアクション戦闘、グライダー移動、探索の楽しさ
- **どうぶつの森**: 見下ろし気味のカメラアングル、穏やかな世界の雰囲気

**明確に入れないもの**:
- レッドストーン的な回路/論理システム
- ハードコアサバイバル（飢餓ゲージ等は未定、プロトタイプでは入れない）
- PvP戦闘（プロトタイプ段階では）

## 2. 技術スタック

| カテゴリ | 技術 | 理由 |
|---------|------|------|
| レンダリング | Three.js | Claudeの生成精度が最も高いWebGL系ライブラリ |
| 言語 | TypeScript | 型があることでClaude Codeの生成精度が向上する |
| ビルド | Vite | HMRが速くイテレーション向き |
| マルチプレイ | Colyseus + Node.js | WebSocketベース、TypeScriptで統一可能 |
| iPad展開 | Capacitor | Web→ネイティブラッパー、段階的に移行可能 |
| ノイズ生成 | simplex-noise (npm) | 地形生成用。自作しない |
| 物理/衝突 | cannon-es (npm) | 物理演算。自作しない |

**vibe codingルール**:
- 既存npmパッケージで済む機能は必ずパッケージを使う（自作しない）
- マジックナンバーはすべて `constants.ts` に集約する

## 3. カメラシステム

**タイプ**: 3rdパーソン、デフォルトでは見下ろし気味のカメラ

**数値仕様**:
- カメラ角度: 地面に対して約45〜55度の俯角（どうぶつの森的な見下ろし感）
- カメラ距離: プレイヤーの後方上空12〜18ブロック分
- FOV: 60度
- カメラはプレイヤーを常に追従する（スムーズ補間あり、lerp係数: 0.1）
- 右スティック（またはマウスドラッグ）でカメラ水平回転・上下回転が可能
- カメラ縦角度の範囲: 10度（ほぼ水平）〜 80度（ほぼ真上）

**制約**:
- カメラとプレイヤーの間にブロックがある場合、そのブロックを半透明にする（カメラを近づけない）
- 浮島の端でカメラが宙に飛び出しても問題ない（空が見える世界なので自然）

## 4. プレイヤーキャラクター

**見た目**: Minecraftスキン的なボクセルキャラクター
- 頭: 8×8×8 ボクセル
- 体: 8×12×4 ボクセル
- 腕: 4×12×4 ボクセル × 2
- 脚: 4×12×4 ボクセル × 2
- 全高: 約2ブロック分

**移動**:
- 歩行速度: 4ブロック/秒
- 走行速度: 7ブロック/秒（走行ボタン長押し）
- ジャンプ高さ: 1.25ブロック分（1ブロックの段差を超えられる）
- 重力加速度: 20ブロック/秒²
- 水平方向の加速/減速あり（即座に最高速にならない）

**グライダー**:
- ジャンプ中にグライダーボタンで展開
- 滑空中は緩やかに下降（下降速度: 2ブロック/秒）
- 前方への水平速度: 8ブロック/秒
- 上昇気流エリアに入ると上昇可能（上昇速度: 4ブロック/秒）
- スタミナ制限あり（最大10秒間、地面に着くと回復）

## 5. ワールド生成

**ワールド構造**:
- 浮島ベースの世界。地面（Y=0）は存在しない
- 浮島はY=20〜Y=100の範囲に生成される
- 島のサイズ: 小（半径8〜15ブロック）、中（半径20〜40ブロック）、大（半径50〜80ブロック）
- 島と島の間隔: 最小10ブロック、最大50ブロック
- 奈落（Y=-10以下）に落ちたらダメージ＋最後にいた島にリスポーン

**チャンクシステム**:
- チャンクサイズ: 16×16×16 ブロック
- 描画距離: 8チャンク（128ブロック）※パフォーマンスに応じて調整
- プレイヤーの周囲のチャンクのみロード/メッシュ生成
- Greedy Meshingで面を結合し、描画ポリゴンを削減

**バイオーム** (プロトタイプで実装する3種):

| バイオーム | 地表ブロック | 特徴 | 木/植生 | 敵の種類 |
|-----------|------------|------|--------|---------|
| 草原 | 草ブロック | なだらかな丘陵 | オークの木 | スライム |
| 森 | 草ブロック（暗） | 密集した大木 | 巨大樹 | ゴブリン |
| 砂漠 | 砂ブロック | 平坦、サボテン | サボテン、枯れ木 | サソリ |

**地形生成アルゴリズム**:
- Simplex Noiseの2Dオクターブ（4層）で島の形状を決定
- 別のノイズレイヤーでバイオームを割り当て
- 島の「浮遊感」は3Dノイズで底面を不規則に削る

## 6. ブロックシステム

**ブロックサイズ**: 1×1×1（ワールド座標の1単位 = 1ブロック）

**プロトタイプで実装するブロック種**:

| ID | 名前 | 硬度（破壊時間秒） | ドロップ | 備考 |
|----|------|-------------------|---------|------|
| 0 | 空気 | - | - | 透明 |
| 1 | 土 | 0.5 | 土 | |
| 2 | 草 | 0.5 | 土 | 上面がテクスチャ違い |
| 3 | 石 | 1.5 | 石 | |
| 4 | 木の幹 | 1.0 | 木材 | |
| 5 | 葉 | 0.3 | たまに苗木 | 半透明 |
| 6 | 砂 | 0.5 | 砂 | 重力影響あり（落下する） |
| 7 | 木材（クラフト） | 1.0 | 木材 | 建材用 |
| 8 | 石レンガ（クラフト） | 2.0 | 石レンガ | 建材用 |
| 9 | クラフト台 | 1.0 | クラフト台 | インタラクト可能 |

**ブロック設置/破壊の操作**（見下ろしカメラ用）:
- カーソル（マウス or タッチ）でブロック位置を指定
- レイキャストでカーソル位置からブロックを特定
- 左クリック/タップ長押し → ブロック破壊（硬度に応じた時間）
- 右クリック/タップ → ブロック設置（対象ブロックの隣接面）
- 設置時に「どの面に設置するか」はレイキャストのhit normalで決定
- 設置範囲: プレイヤーから半径5ブロック以内
- 破壊範囲: プレイヤーから半径5ブロック以内

## 7. 戦闘システム

**スタイル**: ゼルダ的なリアルタイムアクション

**プレイヤーの戦闘能力**:
- HP: 20（ハート10個 × 2ポイント）
- 攻撃: 剣を振る（前方90度の範囲、リーチ2ブロック）
- 攻撃クールダウン: 0.5秒
- ダメージ: 装備武器に依存（木の剣=3、石の剣=5）
- 被ダメージ時のノックバック: 2ブロック分
- 被ダメージ後の無敵時間: 1.0秒
- 回避アクション: 横ステップ（ダッシュ方向に3ブロック瞬間移動、クールダウン1.5秒）

**敵（モブ）の基本AI**:
- 平時: ランダムウォーク（待機3秒→移動5ブロック→待機）
- 感知範囲: プレイヤーから10ブロック以内で敵対化
- 敵対時: プレイヤーに向かって直進、攻撃範囲に入ったら攻撃
- 経路探索: 単純な直線移動 + 1ブロックの段差は自動ジャンプ（A*は使わない）

**昼夜サイクルとの連動**:
- 1日 = リアル20分（昼12分、夜8分）
- 夜間: 敵のスポーン率2倍、敵のHP 1.5倍
- 夜間限定の強敵（ファントム等）が出現

## 8. クラフトシステム

**仕組み**:
- フィールド上でレシピアイテムを発見すると、そのレシピが恒久的にアンロックされる
- レシピを持っている＋素材が揃っている → どこでも即座にクラフト可能（クラフト台不要）
- クラフト台は特別なレシピ（高度な装備等）にのみ必要

**プロトタイプのレシピ例**:

| 成果物 | 素材 | レシピ入手場所 |
|--------|------|---------------|
| 木の剣 | 木材×3 | 初期所持 |
| 石の剣 | 石×4、木材×1 | 草原の島に配置 |
| 木の盾 | 木材×5 | 森の島に配置 |
| グライダー | 木材×3、葉×5 | 最初の浮島（チュートリアル） |
| 包帯 | 葉×3 | 初期所持（HP5回復） |

## 10. マルチプレイ

**方式**: リアルタイム同期（Colyseus WebSocket）

**プロトタイプ段階の仕様**:
- 最大4人の協力プレイ
- ホスト1人がサーバーを立てる（Colyseus）
- 同じワールドを共有、各プレイヤーの位置・アクション同期
- ブロック変更はサーバー経由で全員に同期
- 敵のスポーン/AIはサーバー側で処理

**同期する情報**:
- プレイヤー位置、向き、アニメーション状態
- ブロックの設置/破壊
- 敵の位置、HP
- チャットメッセージ（テキストのみ）

**同期しない情報（各クライアントのローカル）**:
- 個人のインベントリ
- カメラ位置
- パーティクルエフェクト

## 13. Claude Code向けの注意事項

1. **座標系**: Three.jsデフォルト（Y-up）を使用。Yが高さ、XZが水平面。
2. **1ブロック = 1ユニット**: Three.jsのワールド座標1.0 = 1ブロック。
3. **チャンクの原点**: チャンクのワールド座標 = `(chunkX * 16, chunkY * 16, chunkZ * 16)`。
4. **テクスチャ**: プロトタイプではプロシージャル生成（Canvas API）で色分けするだけでOK。画像テクスチャは後で差し替え可能にする。
5. **パフォーマンス目標**: 60fps（デスクトップChrome）。iPad Safariでは30fps以上を目指す。
6. **エラーハンドリング**: ワールド範囲外のブロックアクセスは常にブロックID 0（空気）を返す。
7. **セーブ**: プロトタイプではブラウザのIndexedDBに保存。サーバーサイドセーブはPhase 7以降。
