# PLAN - やりたいこと

- ボクセルベースの浮島探索ゲーム「CCCraft」のWebプロトタイプ。

## 開発方針

### 全体原則
- **vibe codingプロジェクト**: ユーザーはコードを直接書かない。Claude Codeがすべて実装する。
- **GDD.mdが正**: ゲーム仕様に関する判断はGDD.mdに従う。GDD.mdに記載がない場合は、ユーザーに確認する前に最も一般的な実装を選ぶ。
- **段階的開発**: Phase 0→1→2→...の順番を守る。Phaseを飛ばさない。

### コーディング規約
- **TypeScript strict mode** を使用する
- **マジックナンバー禁止**: 数値は全て `src/constants.ts` に定義する
- **npmパッケージ優先**: アルゴリズムを自前実装する前に、既存パッケージがないか確認する
- **コメントは日本語** で書く

### Three.js固有のルール
- 座標系: Y-up（Three.jsデフォルト）
- 1ワールドユニット = 1ブロック
- チャンクサイズ: 16×16×16
- レンダラー: WebGLRenderer（WebGPURendererは使わない）
- テクスチャ: プロトタイプではCanvas APIでプロシージャル生成。画像ファイルは使わない。

### ディレクトリ構造
GDD.mdの「プロジェクト構造」セクションに従う。新しいファイルを追加する場合は、最も近い既存ディレクトリに入れる。
