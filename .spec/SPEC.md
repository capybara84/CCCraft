# SPEC - Phase 1: ワールドとキャラクター

## 概要
フラットな浮島1つの上を、ボクセルキャラクターが歩き回れるようにする。

## 機能要件

### 1. フラット浮島の描画
- 中サイズの浮島（半径30ブロック程度）を1つ描画
- 浮島はY=40付近に配置（GDD: Y=20〜100の範囲）
- 地表は草ブロック（上面緑、側面茶色）、その下に数層の土ブロック
- 島の厚さは5ブロック程度（草1層 + 土4層）
- テクスチャはCanvas APIでプロシージャル生成（画像ファイル不使用）

### 2. ボクセルキャラクター
- GDD準拠のサイズ（全高約2ブロック）:
  - 頭: 8×8×8 ボクセル（=1×1×1ブロック）
  - 体: 8×12×4 ボクセル（=1×1.5×0.5ブロック）
  - 腕: 4×12×4 ボクセル（=0.5×1.5×0.5ブロック）× 2
  - 脚: 4×12×4 ボクセル（=0.5×1.5×0.5ブロック）× 2
- Three.jsのBoxGeometryで各パーツを構成
- プロトタイプカラー（肌色の頭、青い体、肌色の腕、濃紺の脚）
- **歩行アニメーション**: 腕と脚を前後に振る（sin波ベース）

### 3. 3rdパーソンカメラ
- GDD準拠:
  - 俯角: 45〜55度（固定、プレイヤー操作不可）
  - 距離: 12〜18ブロック（初期値15）
  - FOV: 60度
  - スムーズ追従（lerp係数0.1）
- マウスドラッグ（右ドラッグまたは左ドラッグ）で水平回転
- 縦角度は固定

### 4. プレイヤー移動（WASD）
- GDD準拠の移動パラメータ:
  - 歩行速度: 4ブロック/秒
  - 走行速度: 7ブロック/秒（Shift長押し）
  - 水平方向の加速/減速あり
- カメラの向きに対して相対的に移動（W=カメラ前方）

### 5. ジャンプ + 重力（cannon-es）
- cannon-esで物理演算:
  - 重力加速度: 20ブロック/秒²
  - ジャンプ高さ: 1.25ブロック
- プレイヤーの衝突体: カプセルまたはボックス（全高2ブロック）
- 浮島の衝突体: 静的なボックス群またはトライメッシュ
- 接地判定あり（空中ではジャンプ不可）

## 非機能要件
- TypeScript strict mode
- マジックナンバーは全て constants.ts に定義
- コメントは日本語
- 60fps維持（デスクトップChrome）

## 作成・変更するファイル

```
src/
├── main.ts              ← Game初期化（変更）
├── constants.ts         ← Phase 1パラメータ追加（変更）
├── Game.ts              ← シーン管理、各コンポーネント統合（変更）
├── world/
│   └── FlatIsland.ts    ← フラット浮島の生成・描画
├── player/
│   ├── PlayerController.ts  ← 移動・ジャンプ制御（cannon-es連携）
│   ├── PlayerModel.ts       ← ボクセルキャラクターモデル・アニメーション
│   ├── CameraController.ts  ← 3rdパーソンカメラ
│   └── InputManager.ts      ← キーボード・マウス入力管理
└── blocks/
    └── BlockTextures.ts     ← プロシージャルテクスチャ生成
```

## 新規依存パッケージ
- `cannon-es`: 物理演算

## 完了基準
- 1つの浮島の上をボクセルキャラクターがWASD+Shiftで歩き/走りできる
- マウスドラッグでカメラが水平回転する
- スペースキーでジャンプでき、重力で着地する
- 島の端から落ちると重力で落下する
- 歩行時に腕と脚が振れるアニメーションが再生される
